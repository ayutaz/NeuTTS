# 関連研究と本研究の位置付け

## 1. コーデックベース音声合成の不安定性への既存アプローチ

コーデックベースTTS（例: VALL-E系）は高品質な音声合成を実現する一方で、繰り返し退化・省略・不自然な発話といった不安定性が問題となっている。既存のアプローチは大きく2つのカテゴリに分類できる。

### カテゴリA: 追加訓練・最適化ベース

訓練プロセス自体を改善することで不安定性を軽減する手法群である。

- **SpeechAlign** [11]: golden（実音声由来）トークンとsyntheticトークン間のmismatchをpreference-based optimizationによって削減する。
- **Human feedback** [12]: 知覚品質向上のために人間フィードバック信号を訓練に活用する。
- **Differentiable reward** [13]: 微分可能な報酬信号を最適化目標に直接統合し、品質指標をエンドツーエンドで最適化する。
- **Reverse inference optimization** [14]: 補助的なreverse推論目標を導入することで堅牢性を向上させる。

**問題点**: いずれの手法も再訓練、反復最適化、またはキュレーションされたデータが必要であり、計算コストとシステム複雑性の増加を招く。

### カテゴリB: デコーディング時調整

訓練済みモデルのパラメータを変更せず、推論時のデコーディング戦略を工夫する手法群である。

- **VALL-E 2** [16]: repetition-aware samplingを導入し、自己回帰デコーディング中の繰り返し退化を軽減する。繰り返し制御とサンプリング戦略の変更を組み合わせている。
- **ELLA-V** [7]: alignment-guided sequence reorderingにより、long-contextおよびzero-shot設定における安定性を向上させる。アライメント制約を活用した手法である。
- **VALL-E R** [17]: inference-timeにおけるalignment constraintsとdecoding refinementsを組み合わせ、生成品質を改善する。

**問題点**: これらの手法は特定の失敗パターン（繰り返し、アライメント崩壊など）を対象とするが、生成されたトークン列が全体的に一貫しているか、あるいは局所的に自然であるかを明示的に評価する仕組みを持たない。

## 2. Controllable Text Generationとの関連

自然言語処理分野では、外部信号を用いてデコーディングを制御する手法が研究されてきた。

- **Dexperts** [18]: 外部分類器（expert/anti-expert）をデコーディング中に組み込み、トークンレベルの確率分布を調整する。
- **Plug and Play LM** [19]: 言語モデルの出力を外部の評価信号（属性分類器など）でガイドし、所望の属性を持つテキストを生成する。

これらの手法は、強化学習によってジェネレーター自体を最適化する代わりに、デコーディング中に明示的な評価メカニズムを適用するアプローチを採る。MSpoof-TTSはこの考え方を音声合成に応用し、スプーフ検出モデルを外部分類器として使用することで、デコーディング中にauthenticityスコアに基づいて候補トークン列を選別する。

## 3. 音声スプーフ検出 (Anti-Spoofing)

合成音声や改ざん音声を検出する研究は、音声セキュリティ分野で広く取り組まれてきた。

- **従来の研究** [20-24]: 連続音声ドメイン（波形レベル）での合成・改ざん音声検出手法。
- **ASVspoof challenge** [25-27]: 波形レベルの偽造音声検出ベンチマークとして、分野の発展を牽引してきた。
- **コーデックベースdeepfake** [29-31]: Codecfake、Codecfake+、Codecfake-omniといった研究は、コーデックベースの音声合成がもたらす新しい偽造音声の課題に取り組んでいる。

### 既存手法の限界

従来のスプーフ検出手法には以下の限界がある。

- 主に再構成（デコード後）された音声信号上で定式化されている。
- 事後的（post-hoc）な分類を目的として設計されている。
- 離散コーデックトークン列上では直接動作しない。
- 生成プロセス中のデコーディングガイダンスとしては設計されていない。

## 4. MSpoof-TTSの新規性

上記の関連研究を踏まえ、MSpoof-TTSは以下の4つの点で新規性を有する。

| 観点 | 従来手法 | MSpoof-TTS |
|------|----------|------------|
| 評価対象 | 波形（連続信号） | 離散コーデックトークン列 |
| 評価粒度 | 単一解像度 | マルチ解像度（複数時間スケール） |
| 適用タイミング | 事後的（post-hoc）分類 | 生成プロセス中のリアルタイムガイダンス |
| TTSモデルへの影響 | 再訓練・パラメータ変更が必要な場合あり | Training-free（ベースTTSのパラメータ不変） |

- **トークンレベルでのスプーフ検出**: 波形ではなく離散コーデックトークン列を直接評価することで、デコーディング段階での介入を可能にする。
- **マルチ解像度評価**: 複数の時間スケールでauthenticityを評価し、局所的な不自然さと全体的な一貫性の両方を捉える。
- **デコーディング統合**: post-hoc分類ではなく、生成プロセス中にリアルタイムでガイダンスを提供する。Controllable Text Generationの知見を音声合成に転用したアプローチである。
- **Training-free（TTS側）**: ベースTTSモデルのパラメータを一切変更しないため、任意のコーデックベースTTSにプラグインとして適用可能である。

## 実装における関連技術の活用

本プロジェクトの実装において、論文で言及された関連技術は以下のように活用されている。

### Controllable Text Generation の適用

論文で言及された Dexperts や Plug and Play LM の考え方は、`HierarchicalDecoder` の設計に反映されている。スプーフ検出器が外部分類器として機能し、デコーディング中にauthenticityスコアに基づいて候補を選別する。

- 実装: `mspoof_tts/sampling/hierarchical.py` の `_stage1_prune()`, `_stage2_prune()`, `_final_select()`

### RAS から EAS への改良

VALL-E 2 の Repetition-Aware Sampling を改良した Entropy-Aware Sampling は、以下の点で実装に反映：

- ランクベースの重み付け: `MemoryBuffer.get_penalty()` の `1.0 / (1.0 + ranks)` 項
- 指数的時間減衰: `beta ** ages` 項
- クリッピング: `torch.clamp(penalty, max=gamma)` 項
- 実装: `mspoof_tts/sampling/eas.py`

### 音声スプーフ検出技術の転用

従来は事後的な分類に使われていたスプーフ検出を、離散トークン列に直接適用しデコーディングをガイドする新規アプローチ。Conformerアーキテクチャの局所的特徴抽出と大域的依存性捕捉の両方の特性を活用。

- 実装: `mspoof_tts/models/conformer.py`, `spoof_detector.py`, `multi_resolution.py`
